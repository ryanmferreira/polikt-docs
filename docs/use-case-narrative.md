# Narrativas de Caso de Uso - Polikt, Consciência Política e Cidadania

### UC01 - Login / Signup
| Campo                       | Descrição                                                                                                                                                                                                                                                                                                                            |
| :-------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Atores**                  | Usuário Autenticado e Administrador                                                                                                                                                                                                                                                                                                  |
| **Requisito Relacionado**   | RF07 - Perfil do Usuário (Controle de Acesso)                                                                                                                                                                                                                                                                                        |
| **Resumo**                  | Permite ao cidadão ou administrador criar uma conta ou realizar a autenticação para acessar funcionalidades restritas.                                                                                                                                                                                                               |
| **Pré-condições**           | O usuário deve possuir um dispositivo conectado à internet.                                                                                                                                                                                                                                                                          |
| **Pós-condições**           | O ator é identificado pelo sistema e ganha acesso às suas respectivas permissões de uso.                                                                                                                                                                                                                                             |
| **Fluxo Principal**         | 1. O usuário acessa a tela de autenticação do sistema.<br>2. O usuário escolhe entre realizar um novo cadastro ou entrar.<br>3. O usuário insere suas credenciais de acesso.<br>4. O sistema valida as informações inseridas no banco de dados.<br>5. O sistema redireciona o ator para a tela inicial correspondente ao seu perfil. |
| **Exceções / Alternativos** | **FE01 - Credenciais Inválidas:** No passo 4, se os dados estiverem incorretos, o sistema exibe uma mensagem de erro e solicita a correção, retornando ao passo 3.                                                                                                                                                                   |

### UC02 - Feed Personalizado
| Campo                       | Descrição                                                                                                                                                                                                                                                                                                                     |
| :-------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Ator Principal**          | Usuário Autenticado                                                                                                                                                                                                                                                                                                           |
| **Requisito Relacionado**   | RF01 - Feed Personalizado                                                                                                                                                                                                                                                                                                     |
| **Resumo**                  | Exibe notícias e atualizações filtradas de acordo com a localização e preferências de temas cadastrados pelo cidadão.                                                                                                                                                                                                         |
| **Pré-condições**           | O usuário deve estar autenticado no sistema.                                                                                                                                                                                                                                                                                  |
| **Pós-condições**           | O usuário visualiza as publicações pertinentes à sua região.                                                                                                                                                                                                                                                                  |
| **Fluxo Principal**         | 1. O usuário acessa a tela principal do aplicativo.<br>2. O sistema consulta as preferências de temas e a localização configuradas no perfil do usuário.<br>3. O sistema filtra e recupera as notícias correspondentes à região e interesses.<br>4. O sistema renderiza o feed estruturado na tela inicial do usuário.        |
| **Exceções / Alternativos** | **FA01 - Extensão Notícias "Traduzidas" (`<<extend>>`):** No passo 4, o usuário pode selecionar uma notícia específica para ler sua versão simplificada (UC03).<br><br>**FA02 - Extensão Avaliar Clareza (`<<extend>>`):** Durante a leitura do feed, o usuário pode registrar sua percepção sobre a clareza do texto (UC04). |

| **Requisito Relacionado**   | RF02 - Tradutor de Notícias                                                                                                                                                                                                                                 |

| **Fluxo Principal**         | 1. O usuário clica em uma notícia em destaque no seu feed.<br>2. O sistema carrega o texto original e a sua respectiva "tradução" simplificada.<br>3. O sistema exibe o resumo didático ressaltando os impactos práticos daquela medida na vida do cidadão. |

### UC07 - Denúncias / Reclamações da Infraestrutura

| **Atores**                  | Usuário Autenticado e Administrador / Órgão                                                                                                                                                                                                                                                                                                                                                           |
| **Requisito Relacionado**   | RF04 - Mural de Infraestrutura                                                                                                                                                                                                                                                                                                                                                                        |
| **Resumo**                  | Permite o registro público de falhas em zeladorias urbanas no mural comunitário, utilizando fotos e geolocalização.                                                                                                                                                                                                                                                                                   |

| **Fluxo Principal**         | 1. O usuário acessa o Mural de Infraestrutura do aplicativo.<br>2. O usuário opta por criar um novo relato sobre um problema urbano presenciado.<br>3. O sistema solicita e os dados.<br>4. O usuário insere uma descrição detalhada do problema e anexa uma foto comprovatória.<br>5. O usuário confirma o envio da postagem.<br>6. O sistema armazena a ocorrência e a exibe publicamente no mural. |

### UC09 - Guia de Denúncia
| Campo                       | Descrição                                                                                                                                                                                                                                                                                                                                                            |
| :-------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Ator Principal**          | Usuário Autenticado                                                                                                                                                                                                                                                                                                                                                  |
| **Requisito Relacionado**   | RF05 - Guia de Denúncia                                                                                                                                                                                                                                                                                                                                              |
| **Resumo**                  | Orienta o cidadão através de um manual prático sobre como formalizar reclamações oficiais de infraestrutura perante os canais da administração pública.                                                                                                                                                                                                              |
| **Pré-condições**           | O usuário deve ter selecionado um problema de infraestrutura específico.                                                                                                                                                                                                                                                                                             |
| **Pós-condições**           | O usuário adquire o conhecimento formal necessário para procurar as ouvidorias oficiais competentes.                                                                                                                                                                                                                                                                 |
| **Fluxo Principal**         | 1. O usuário visualiza uma ocorrência de infraestrutura no aplicativo.<br>2. O usuário seleciona o botão para abrir o "Guia de Denúncia" correspondente àquela categoria de falha.<br>3. O sistema exibe um passo a passo objetivo contendo as instruções, leis aplicáveis e os canais corretos (sites, telefones ou endereços) para registrar a reclamação oficial. |
| **Exceções / Alternativos** | Nenhum.                                                                                                                                                                                                                                                                                                                                                              |
